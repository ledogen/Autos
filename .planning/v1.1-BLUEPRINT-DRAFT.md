## Goal

An Eastern-Sierra-style landscape — flat valleys, steep mountain faces — with
procedurally-routed roads that switchback up grades too steep to climb straight,
giving the player a smooth preferred surface in an otherwise lively off-road
world. Plus a dev free-cam to evaluate it, a world-seed system for reproducible
maps, and a clean seam for future POI/mission content.

Scope intent: **slightly less ambitious than v1.0** — 4 phases (foldable to 3).

---

## ⊕ Foundation — World Seed (established in Phase 7)

- One `worldSeed` drives ALL procedural generation. Same seed ⇒ byte-identical
  world: terrain, roads, POIs. Different seed ⇒ different but fully reproducible
  world. Seeds are shareable.
- **Sub-seed derivation:** each system pulls an independent stream by hashing the
  world seed with a domain tag — `seedFor("coarse")`, `seedFor("fine")`,
  `seedFor("regional")`, `seedFor("roads", tileX, tileZ)`, `seedFor("poi", splineId)`
  — so terrain noise and road/POI placement don't correlate.
- **HARD RULE:** every generator is a pure function of `(worldSeed, world coords)`.
  NO dependence on chunk load order, frame timing, or visit history. This is what
  makes maps replicable — and it's the same constraint the deterministic road
  tile-graph needs.
- **Surfacing:** seed shown + editable in the debug panel, plus a `?seed=...` URL
  param for shareable maps (zero-infra, fits GitHub-Pages / no-backend). Refresh
  with same seed = same world.
- **Seed format (plan-time detail):** accept a string ("lone-pine-1") hashed to a
  32-bit int, or a raw int. Friendlier for sharing.
- **Verify at plan time:** confirm the current simplex source (terrain.js) accepts
  a seed / permutation table. If it's fixed-permutation today, P7 adds the hook.

## ① Layered Terrain — Phase 7

- **Coarse layer:** large-scale simplex tuned to Eastern Sierra GRADES — steep
  faces, flat valley floors. Amplitude outpaces wavelength (steep, NOT gentle).
  Likely multi-octave / ridged rather than one smooth octave to get escarpment
  character.
- **Fine layer:** today's-scale noise added on top → suspension texture for
  lively, harder-to-control off-road driving.
- **Regional roughness:** low-frequency multiplier field on the fine layer only
  (random now; difficulty-system hook later).
- **Unified `height(x,z)`:** ONE pure function called by both the Web Worker mesh
  build and the physics sampler. Disagreement = truck floats/sinks. **Milestone's
  #1 correctness constraint.**
- **Dev FREE CAM:** add a free-fly mode to `camera.js` (toggle key, fly with
  WASD + look + vertical, decoupled from the truck, car idles while flying).
  `camera.js` was architected for multi-mode, so this adds a mode, not a rewrite.
  Delivered FIRST in P7 so the rest of terrain tuning is observable.
- **Topo baseline:** calibrate coarse amplitude/wavelength/octaves to match real
  Eastern Sierra topo statistics (user to provide a topo/DEM, e.g. Lone Pine /
  Mt. Whitney quad). Match the *statistics* of real terrain — do NOT import a
  finite DEM; the world stays infinite & procedural. Topo doubles as a visual
  validation target.

## ② Road Routing → Splines — Phase 8

- Deterministic, tile-able road graph routed over COARSE height only (meanders
  with mountains, ignores fine texture), seeded via `seedFor("roads", …)`.
- Slope-weighted cost + hard max-grade limit. Because terrain is genuinely steep,
  the router must **SWITCHBACK** where a direct line exceeds max grade — not just
  gently wrap. This is the meatier, riskier half of the milestone.
- Output: queryable splines (debug-line visualized; no mesh yet), deterministic &
  seamless across streaming chunks via a chunk-keyed spatial index.

## ③ Road Surface — Phase 9

- ~10 m fixed-width ribbon swept along splines.
- Cross-section: 2–3° crown at centerline + curvature-driven camber that banks
  into turns.
- **Physics surface NORMAL must carry crown + bank**, or the car won't feel them.
- **Carve:** road suppresses fine noise & flattens terrain toward the road
  surface, blended over a shoulder width — applied identically in mesh + physics
  sampler (same coupling risk as the unified height fn).

## ④ POI Anchor Hooks + Polish — Phase 10

- POI hooks: emit `{position, road tangent, type slot}` anchors at low-slope,
  road-adjacent sites, placed via `seedFor("poi", …)` → same gas station in the
  same spot for a given seed. **Data contract only — no spawning, no models.**
  Future gameplay phase consumes it (gas station / parking lot / trailer home →
  mission pickups).
- **Stretch:** pothole / crack micro-noise that perturbs only the road surface.
