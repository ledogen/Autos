Vehicle state shape (after Plan 02):
  vehicleState.strutComp[4]    : m  — positive = compressed below rest (D-01)
  vehicleState.strutCompVel[4] : m/s — positive = compressing (D-03)
  (hubY / hubVy removed entirely)

Suspension accumulator contract:
  params._tireFz[i]         : scalar — strut-axis projection of tire normal force (D-06b)
  params._suspForceAccum[i] : scalar — magnitude of spring + damper force on body (applied along body-up, D-04)
  params._hubNormalXZ[i]    : {x,y,z} plain object — X/Z residual contact normal force, applied at hub world position with torque arm (D-06a)

Pure-math contract for suspension.js:
  MUST NOT import Three.js.
  Use params._rotateVector({x,y,z}) which returns a plain {x,y,z} object.
  Manual dot product: a.x*b.x + a.y*b.y + a.z*b.z (not THREE.Vector3.dot()).

Strut-axis math (D-02, D-03):
  body_down = params._rotateVector({x:0, y:-1, z:0})
  body_up   = params._rotateVector({x:0, y: 1, z:0})
  strutLen  = L_S - strutComp[i]
  hubWorld  = mountWorld + strutLen * body_down       (component-wise)
  mountVelStrut = dot(mountVel3D, body_down)
  strutCompVel  = mountVelStrut - hubVelAlongStrut   (small-mass approx: hubVelAlongStrut ~= 0)

Contact normal split (D-06):
  bodyUpDot = dot(c.normal, body_up)
  Fn_strut  = tireFnAtContact * bodyUpDot                      -> tireFz, drives hub ODE
  Fn_XZ     = tireFnAtContact * (c.normal - bodyUpDot * body_up)  -> _hubNormalXZ accumulator

Stop forces (D-08, D-09):
  bumpOvershoot  = strutComp - travel
  bumpForce      = bumpOvershoot > 0 ? -bumpStopStiffness * bumpOvershoot : 0
  droopOvershoot = -strutComp
  droopForce     = droopOvershoot > 0 ?  DROOP_STOP_STIFFNESS * droopOvershoot : 0
