    GLOSSARY.md additions (Phase 6 section):
    - chunk: 64×64-unit tile of heightmap terrain; backed by a 65×65 Float32Array of heights; identified by integer grid coordinates (cx, cz); generated by the TerrainSystem Web Worker
    - heightmap: a 2D grid of height values (Float32Array) representing terrain surface elevation; cell size 1.0 m; bilinear interpolated for physics queries
    - bilinear interpolation: O(1) height lookup within a cell using four corner heights weighted by fractional position; used by sampleHeight() in TerrainSystem
    - terrainAmplitude: RANGER_PARAMS field (float, default 1.0); scale multiplier applied to raw noise heights during chunk geometry build; higher values produce steeper hills; tunable via Terrain debug folder
    - chunk ring: the 5×5 set of chunks surrounding the car (RING_RADIUS=2); maintained by TerrainSystem.update(); chunks outside the ring are disposed; chunks inside not yet loaded are requested from the worker
  </behavior>
