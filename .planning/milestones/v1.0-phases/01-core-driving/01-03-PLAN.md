    Create src/vehicle.js. This module manages keyboard input state and computes per-step vehicle control values. It does NOT import Three.js — all geometry is handled by physics.js. It exports updateVehicle (called once per fixed step before stepPhysics) and SPAWN_STATE (for the R-key reset).

    File header comment: "Vehicle input and control module. Owns keyboard state, steer accumulation, Ackermann geometry, wheel spin accumulation, and spawn-reset. Does NOT perform physics integration — that is physics.js. Does NOT import Three.js. Pure math + DOM event listeners."

    Module-level input state (module-private, not exported):
      const keys = { w: false, s: false, a: false, d: false, r: false }
    Register keydown/keyup listeners immediately at module load (not gated on DOMContentLoaded — module scripts run after parse):
      document.addEventListener('keydown', e => { const k = e.key.toLowerCase(); if (k in keys) keys[k] = true })
      document.addEventListener('keyup',   e => { const k = e.key.toLowerCase(); if (k in keys) keys[k] = false })

    Export — SPAWN_STATE (plain object, not THREE objects — main.js will copy values):
      {
        positionX: 0, positionY: (caller uses RANGER_PARAMS.cgHeight), positionZ: 0,
        velocityX: 0, velocityY: 0, velocityZ: 0,
        quatX: 0, quatY: 0, quatZ: 0, quatW: 1,           // identity quaternion
        angVelX: 0, angVelY: 0, angVelZ: 0,
        steerAngle: 0, throttle: 0, brake: 0,
        wheelAngles: [0, 0, 0, 0],
        wheelSteerAngles: [0, 0, 0, 0]
      }
    Comment above export: "Plain scalar values — main.js copies these into THREE.Vector3 / THREE.Quaternion fields of vehicleState on R-key reset."

    Export — updateVehicle(vehicleState, params, dt):
    JSDoc:
      @param {object} vehicleState — mutable vehicleState shape from main.js; mutated in-place
      @param {object} params — RANGER_PARAMS (may have debug-slider overrides)
      @param {number} dt — fixed timestep in seconds (1/60)
      @returns {boolean} true if R-key reset was requested this step (main.js handles the copy)
    Implementation in order:

    1. Throttle / Brake (M1-05):
       vehicleState.throttle = keys.w ? 1 : 0
       vehicleState.brake    = keys.s ? 1 : 0

    2. Speed-scaled steer limit (M1-08):
       Compute current speed in m/s: const speed = Math.sqrt(vehicleState.velocity.x**2 + vehicleState.velocity.z**2)
       Speed-scaled max steer: const dynamicMaxSteer = params.maxSteerAngle / (1 + speed / params.speedSteerRef)
       Comment: "M1-08: at speed=speedSteerRef (15 m/s ≈ 54 km/h) max steer is halved. Denominator grows linearly with speed."

    3. Steer accumulation (M1-07):
       if (keys.a) vehicleState.steerAngle += params.steerRate * dt
       if (keys.d) vehicleState.steerAngle -= params.steerRate * dt
       if (!keys.a && !keys.d) {
         // Decay toward zero at steerDecayRate
         const decay = params.steerDecayRate * dt
         if (Math.abs(vehicleState.steerAngle) <= decay) {
           vehicleState.steerAngle = 0
         } else {
           vehicleState.steerAngle -= Math.sign(vehicleState.steerAngle) * decay
         }
       }
       Clamp to dynamic max: vehicleState.steerAngle = Math.max(-dynamicMaxSteer, Math.min(dynamicMaxSteer, vehicleState.steerAngle))

    4. Ackermann per-wheel angles (M1-06):
       Use RESEARCH.md §Pattern 5 formula exactly. L = params.wheelbase, T = params.trackFront, phi = vehicleState.steerAngle.
       If Math.abs(phi) < 1e-6:
         wheelSteerAngles = [0, 0, 0, 0]
       Else:
         const sinPhi = Math.sin(phi), cosPhi = Math.cos(phi), twoL = 2 * params.wheelbase
         phiLeft  = Math.atan(twoL * sinPhi / (twoL * cosPhi - params.trackFront * sinPhi))
         phiRight = Math.atan(twoL * sinPhi / (twoL * cosPhi + params.trackFront * sinPhi))
         Positive steerAngle = steer left: left wheel is inner (sharper angle for same-sign phi), right wheel outer.
         wheelSteerAngles = [phiLeft, phiRight, 0, 0]  (rear wheels 2,3 always 0)
       Store on vehicleState: vehicleState.wheelSteerAngles = wheelSteerAngles
       (This field is read by physics.js stepPhysics for lateral force decomposition.)

    5. Wheel visual spin (M1-09):
       All 4 wheels spin at the same rate in Phase 1 (RESEARCH.md §Open Questions #1 — drivetrain split is Phase 2+).
       Longitudinal speed for spin: const longSpeed = vehicleState.velocity.dot-product with forward direction.
       Since vehicle.js has no THREE, compute forward direction as: the negative-Z column of the quaternion.
       Quaternion q = vehicleState.quaternion; forward-Z component:
         const qx=vehicleState.quaternion.x, qy=vehicleState.quaternion.y, qz=vehicleState.quaternion.z, qw=vehicleState.quaternion.w
         const fwdX = 2*(qx*qz + qy*qw)
         const fwdY = 2*(qy*qz - qx*qw)
         const fwdZ = 1 - 2*(qx*qx + qy*qy)
         // forward vector = (fwdX, fwdY, fwdZ) — this is (0,0,-1) rotated by quaternion
         // negate for car forward (-Z body space):
         const longSpeed = -(vehicleState.velocity.x*fwdX + vehicleState.velocity.y*fwdY + vehicleState.velocity.z*fwdZ)
       Spin delta per step: const spinDelta = (longSpeed / params.wheelRadius) * dt
       For each i: vehicleState.wheelAngles[i] += spinDelta
       Comment: "Phase 1: all wheels spin at same rate. Phase 2+ drivetrain will differentiate per-wheel omega."

    6. Reset check (M1-12):
       if (keys.r) { keys.r = false; return true }
       return false
