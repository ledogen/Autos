      import { computeLateralForce, computeLongitudinalForce } from './tire.js'

    Export 1 — getDriveTorque(wheelIndex, vehicleState, params):
    JSDoc:
      @param {number} wheelIndex — 0-3 per GLOSSARY.md wheel index convention
      @param {object} vehicleState — full vehicleState; throttle and brake fields used
      @param {object} params — RANGER_PARAMS; maxDriveTorque and maxBrakeTorque used
      @returns {number} torque [N·m] to apply at this wheel. Positive = drive forward. Phase 1: rear wheels only (RWD — indices 2 and 3) receive drive torque; all wheels receive brake torque. Phase 2+ replaces this with a real drivetrain model.
    Phase 1 body:
      const isRear = wheelIndex === 2 || wheelIndex === 3;
      const driveTorque = isRear ? vehicleState.throttle * params.maxDriveTorque : 0;
      const brakeTorque = -vehicleState.brake * params.maxBrakeTorque;
      return driveTorque + brakeTorque;

    Export 2 — stepPhysics(vehicleState, params, dt):
    JSDoc:
      @param {object} vehicleState — mutable vehicleState object (mutated in-place each step)
      @param {object} params — RANGER_PARAMS (may be augmented with debug-slider values)
      @param {number} dt — fixed timestep in seconds (1/60 from game loop)
      @returns {void} — mutates vehicleState in-place
    Implementation (in order):

    Step 1 — Compute body-space axes from quaternion (do NOT use Euler angles):
      const forward = new THREE.Vector3(0, 0, -1).applyQuaternion(vehicleState.quaternion)
      const right   = new THREE.Vector3(1, 0, 0).applyQuaternion(vehicleState.quaternion)
      const up      = new THREE.Vector3(0, 1, 0).applyQuaternion(vehicleState.quaternion)

    Step 2 — Compute per-wheel forces. For each wheel index i in [0, 1, 2, 3]:

      a. Build a _rotateVector helper for suspension.js (keeps suspension.js Three.js-free):
         params._rotateVector = (v) => new THREE.Vector3(v.x, v.y, v.z).applyQuaternion(vehicleState.quaternion)

      b. Get contact patch world position from suspension:
         const contactPt = getWheelPosition(i, vehicleState, params)
         const contactVec = new THREE.Vector3(contactPt.x, contactPt.y, contactPt.z)

      c. Contact patch velocity = vehicle velocity + (angularVelocity cross (contactPt - CG)):
         const rVec = contactVec.clone().sub(vehicleState.position)
         const contactVel = vehicleState.velocity.clone().add(new THREE.Vector3().crossVectors(vehicleState.angularVelocity, rVec))

      d. Decompose into lateral and longitudinal components in wheel's frame:
         For front wheels (i === 0 or 1), the wheel heading is steerAngle-rotated forward. Use Ackermann
         wheel angle from vehicleState if available (Plan 03 sets vehicleState.wheelSteerAngles[i]);
         if not yet set, fall back to 0 for rear and vehicleState.steerAngle for front.
         Compute wheel forward direction: rotate body forward around body up by per-wheel steer angle.
         const steer = (i < 2 && vehicleState.wheelSteerAngles) ? vehicleState.wheelSteerAngles[i] : (i < 2 ? vehicleState.steerAngle : 0)
         const steerQ = new THREE.Quaternion().setFromAxisAngle(up, steer)
         const wheelFwd  = forward.clone().applyQuaternion(steerQ)
         const wheelRight = right.clone().applyQuaternion(steerQ)
         const longVel = contactVel.dot(wheelFwd)
         const latVel  = contactVel.dot(wheelRight)

      e. Normal force from suspension:
         const Fz = computeNormalForce(i, vehicleState, params)

      f. Drive torque for this wheel → longitudinal drive force:
         const torque = getDriveTorque(i, vehicleState, params)
         const driveForce = torque / params.wheelRadius   // F = T / r

      g. Augment params with per-wheel velocity and drive force for tire.js stubs:
         params._lateralVelocity = latVel
         params._longitudinalVelocity = longVel
         params._driveForce = driveForce

      h. Get tire forces from tire.js (call sites unchanged in Phase 3):
         const Fy = computeLateralForce(0, Fz, params)   // slipAngle unused in Phase 1
         const Fx = computeLongitudinalForce(0, Fz, params)  // slipRatio unused in Phase 1

      i. Accumulate forces and torques in world frame:
         Force in wheel-forward direction: worldForce = wheelFwd.clone().multiplyScalar(Fx)
         Force in wheel-right direction:   worldForce.addScaledVector(wheelRight, Fy)
         Accumulate into totalForce (THREE.Vector3 initialized to zero before the loop).
         Torque from contact offset: worldTorque += rVec.cross(worldForce)
         Accumulate into totalTorque (THREE.Vector3 initialized to zero before the loop).

    Step 3 — Integrate linear velocity and position (Euler):
      vehicleState.velocity.addScaledVector(totalForce, dt / params.mass)
      vehicleState.position.addScaledVector(vehicleState.velocity, dt)

    Step 4 — Integrate angular velocity and orientation:
      vehicleState.angularVelocity.x += totalTorque.x / params.inertiaRoll  * dt
      vehicleState.angularVelocity.y += totalTorque.y / params.inertiaYaw   * dt
      vehicleState.angularVelocity.z += totalTorque.z / params.inertiaPitch * dt
      Call integrateQuaternion (private function, not exported) using RESEARCH.md §Pattern 3 exactly:
        const omega = vehicleState.angularVelocity
        const angSpeed = omega.length()
        if (angSpeed > 1e-10) {
          const axis = omega.clone().normalize()
          const dq = new THREE.Quaternion().setFromAxisAngle(axis, angSpeed * dt)
          vehicleState.quaternion.premultiply(dq).normalize()
        }

    Step 5 — Apply ground constraint (RESEARCH.md §Pattern 7):
      const minY = params.cgHeight
      if (vehicleState.position.y < minY) {
        vehicleState.position.y = minY
        if (vehicleState.velocity.y < 0) vehicleState.velocity.y = 0
      }
      Also zero angular velocity components that would drive the car into the ground when it is on flat terrain (prevents phantom tumbling from numerical errors). When position.y <= minY + 0.01, zero angularVelocity.x and angularVelocity.z: these are pitch and roll rates that a rigid ground would suppress via the normal reaction. Comment: "Phase 4 suspension removes this clamp — spring-damper handles it."

    FORBIDDEN in this file: no bodyMesh.rotation.*, no THREE.Euler for physics state, no references to backup files, no physics library import.
