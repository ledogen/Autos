## Summary

Changed `FWD_THRESHOLD` in `src/physics.js` from `0` to `-2 / 3.6` (-0.556 m/s), mirroring the existing `REV_THRESHOLD = +2/3.6` deadband.

## Root Cause

At rest with W held, numerical noise made `longVel` flip sign every frame between roughly -0.07 and +0.04 m/s. With `FWD_THRESHOLD = 0`, this flipped the input router every frame: drive frame → 800 N·m to rear, brake frame → 3000 N·m brake on all wheels. Rear ω jumped to ~23 rad/s on drive frames and got clamped to 0 on brake frames. Average acceleration ≈ 0 → car stuck at rest under full throttle.

## Fix

`const FWD_THRESHOLD = -2 / 3.6` — W now drives whenever longVel ≥ -2 km/h; brake mode only engages when clearly rolling backward. Symmetric with `REV_THRESHOLD` (S drives reverse until +2 km/h, brakes above).

Also updated stale doc comment that said `(-5 km/h)` (never matched the constant).
