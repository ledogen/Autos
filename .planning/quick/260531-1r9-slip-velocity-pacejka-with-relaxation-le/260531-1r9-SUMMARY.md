## Summary

Two coupled changes that together kill both ω oscillation AND the at-rest lateral oscillation that the previous semi-implicit fix exposed:

1. **Tire model rewrite to slip-VELOCITY Pacejka** (per user direction). Replaces slip ratio + slip angle (which both have singular behavior at zero velocity, producing huge forces from tiny perturbations at rest). Inputs are now slip velocities in m/s. Anisotropy hooks (`tireStiffnessLong`, `tireStiffnessLat`) scale the slip components before computing magnitude — default 1.0 (isotropic).

2. **Per-tire relaxation length** model — physical brush-model dynamics. The tire carcass takes a characteristic distance `L` of vehicle travel to build up to its target force. `ds/dt = v_slip − s·|v|/L` is the spring (build-up) + damper (sliding dissipation) model the user described. Implicit Euler on s is unconditionally stable.

3. **Newton-iterated implicit Euler for ω** (replacing the previous single-step linearized semi-implicit, which over-damped 1000× at rest because `dT/dω → ∞` in the slip-ratio formulation). Newton re-evaluates Pacejka at each iter so it captures saturation past peak — critical for clean launch from rest, where a single linearization would overshoot.

## Why this combination

The previous semi-implicit broke because slip-ratio had `dT/dω → ∞` as v → 0 (the `1/v_eps` SLIP_EPSILON floor produced a *valid* linearization, but only for infinitesimal perturbations — at finite drive torque the linearization is invalid and over-damped 1000×). With slip-velocity Pacejka:
- `dT/dω` is bounded by physical tire stiffness regardless of vehicle speed (no `1/v_eps` blowup)
- The relaxation length further reduces effective `dF/dω` by spreading force build-up over `O(L)` distance
- Together, the implicit denominator stays in the 2–6× range at all speeds → Newton converges in 1–3 iterations

The result is: drive from rest produces wheel ω that smoothly settles into a clean equilibrium tracking the accelerating car (F locked at `T_drive/r` exactly). Coast converges to free-rolling in 3 steps with no overshoot. Brake locks ω → 0 in 2 steps cleanly.

## What was tested (simulated against the production formula)

| Scenario | Behavior |
|---|---|
| Coast (ω=46, vx=15.65, no input) | ω 46 → 44.3 → 43.0 → 42.4 → 42.6 (free-rolling), no oscillation |
| Drive 800 N·m from rest | ω growth smooth, vx grows linearly at 1.4 m/s², F = T_drive/r exactly |
| Hard brake (T=3000) from coast | ω 42.5 → 16 → 0 (locked) in 2 steps; F = -3100 N for monotone car decel |
| Lateral perturbation at rest (latV=0.05) | Damped oscillation bounded at ±0.05, ~20% amplitude decay per ~7-step cycle |

## Known limitations / trade-offs

- **Lateral at rest still oscillates somewhat**: the per-tire relaxation provides "real" carcass damping but not enough to fully critically-damp a 0.05 m/s lateral perturbation at zero forward velocity within one cycle. Amplitude bounded (not growing), envelope decays over ~1-2 seconds. Acceptable for the user's "no low-speed band-aid" requirement. Will be very smooth on a sidehill where `s_lat` builds toward equilibrium against gravity.
- **Single-step transient on first launch frame** — at the very first frame of throttle from rest, ω can overshoot to a small magnitude before Newton finds equilibrium. Visible as a tiny wheel-rotation glitch on the very first frame; subsequent frames are smooth.
- **`pacejkaB/C/D/E` semantics changed**: same Pacejka curve shape, but now interpreted as F vs |s_scaled|/`tireSlipVelRef` rather than vs |slip_ratio_combined|. Initial slope coefficient is `B·C·D` (dimensionless), peak ratio `D`, peak located at |s|/v_ref ≈ 0.1. Slider values may want re-tuning but defaults work.

## Files Changed

- `src/tire.js` — Full rewrite. `computeTireForces(slipVx, slipVy, Fn, params)` returns `{Flong, Flat, dFmagDs}` using slip-velocity combined Pacejka with anisotropy hooks. Analytical `dFmagDs` for Newton iteration; well-defined at slip=0 via limit (`μ·Fn·D·B·C/v_ref`).
- `src/physics.js` —
  - Lazy-init `vehicleState.slipLong[i]`, `slipLat[i]` per tire
  - Inside contacts loop: implicit relaxation update of `sLong`, `sLat`; apply F to body using current ω (operator splitting)
  - ω integrator block: Newton-iterated implicit Euler (max 4 iterations, residual tol 1e-4). Re-evaluates `sLong(ω_new)` and `F(sLong)` each iter to capture Pacejka saturation. Commits final `sLong`.
  - Brake zero-crossing clamp preserved
  - Logger field `_sa` now stores slip-velocity magnitude (m/s) instead of slip-angle (rad) — field name kept for log-format stability
- `data/ranger.js` — Added `tireRelaxationLength` (0.3 m), `tireSlipVelRef` (1.0 m/s), `tireStiffnessLong` (1.0), `tireStiffnessLat` (1.0). Anisotropy hooks ready for tuning to real-tire ratios (~1.5×Long / 1.0×Lat).
- `src/debug.js` —
  - Added sliders for the four new params (relaxation length, slip velocity reference, long/lat stiffness)
  - Updated Pacejka canvas: now plots force magnitude vs slip velocity (0 to 3·v_ref m/s) instead of vs slip angle. Operating-point dot tracks slip velocity magnitude.
- `src/main.js` — Reset path zeros `slipLong`, `slipLat` arrays.

## Tunable parameters & their effect

| Slider | Effect | Typical range |
|---|---|---|
| `Relaxation Length L (m)` | Larger → more damping, slower force response; smaller → snappier, less stable at rest. 0.3 m is standard road tire | 0.05–1.5 |
| `Slip Vel Ref (m/s)` | Slip velocity at peak grip. Lower → reaches peak earlier (snow/rally feel); higher → softer onset | 0.2–5.0 |
| `Stiffness Long ×` / `Stiffness Lat ×` | Scale the slip components before magnitude calc. Real tires have Long ≈ 1.5× Lat; defaults 1.0 (isotropic) | 0.3–2.0 |
| `Friction Coeff` | Global μ multiplier on force magnitude — single grip knob, unchanged behavior | 0.1–1.5 |
| `B/C/D/E` | Pacejka curve shape parameters — unchanged | as before |
