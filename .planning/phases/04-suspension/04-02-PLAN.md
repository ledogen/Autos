<objective>
Replace the matchbox-car model (tire compliance as only spring) with a proper quarter-car at each corner: wheel hubs become integrated rigid bodies with their own vertical position/velocity, a new hub↔body suspension spring sits in series above the existing tire spring, anti-roll bars couple left/right wheels per axle, and the vertical subsystem integrates at dt/2 sub-steps for stability. Pacejka now reads the tire-spring force (per D-03), making wheel-lift detection trivial (tireFz ≤ 0).

This plan delivers M4-01 through M4-07 — the visible physics change. After this plan ships, the user can drive and feel weight transfer (nose dip, body roll, wheels lifting on bumps), but the new sliders, HUD readouts, and visual mesh travel come in Plan 03.

Purpose: All of Phase 4's physics value is in this plan. The split between Task 1 (state schema + params) and Task 2 (math + wiring) lets the schema land first so the math task can reference fully-initialized state without lazy-init hazards (per RESEARCH Pitfall 4).

Output: A driveable build where suspension dynamics, ARBs, and wheel lift are all live, fed by static-equilibrium spawn values that eliminate the first-frame settle.
</objective>
