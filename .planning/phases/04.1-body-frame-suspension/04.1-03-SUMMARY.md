# Phase 4.1 Plan 03: Visual + UI + Logger Surface — Summary

**Wheel mesh D-07 visual confirmed in place; debug panel gains 5 D-14 sliders + 4-corner travel bar; logger schema bumped to 45 fields with fl_sc/fr_sc/rl_sc/rr_sc; GLOSSARY documents strut-axis vocabulary**

## What Was Built

### Task 1: physics.js wheelDebug.strutComp population (D-12)

Added `strutComp: vehicleState.strutComp?.[i] ?? 0` to the `wheelDebug[i]` object at the Step 3 reset (line 209 in physics.js). Uses nullish-safe access so the field is always present even if `strutComp` is undefined. This is the value `captureFrame` reads via `fl.strutComp / rr.strutComp` for the logger.

The D-07 `syncMeshesToState` wheel visual fix (`hubLocal = hubWorld.sub(position).applyQuaternion(carQ.invert())`) was already present from Plan 02 Task 1 sub-change 4. No changes to main.js syncMeshesToState were needed.

### Task 2: debug.js D-14 sliders + D-13 travel bar

Five new sliders added to `suspFolder` after the existing ARB sliders:
- `Front Travel (m)` — `suspensionTravelFront`, range 0.05–0.40 m, step 0.01
- `Rear Travel (m)` — `suspensionTravelRear`, range 0.05–0.40 m, step 0.01
- `Front Body Offset (m)` — `suspensionBodyOffsetFront`, range −0.10 to +0.10 m, step 0.005
- `Rear Body Offset (m)` — `suspensionBodyOffsetRear`, range −0.10 to +0.10 m, step 0.005
- `Bump Stop Stiffness (N/m)` — `bumpStopStiffness`, range 10 000–500 000 N/m, step 5000

`DROOP_STOP_STIFFNESS` deliberately not exposed (fixed constant per CONTEXT.md); noted in a comment above the slider block.

A 160×140 px `travelCanvas` was created and appended to `document.body`. Backtick toggle extended to sync `travelCanvas.style.display` alongside `gui.domElement` and `plotCanvas`.

`updateTravelBars(vehicleState, params)` exported from debug.js:
- Fill formula: `clamp(strutComp[i] / travel[i], 0, 1.05)`
- Color: green (< 70%), yellow (70–95%), red (≥ 100% — bump stop engaged)
- Labels: FL, FR, RL, RR below each bar
- Percentage readout above each fill

### Task 3: logger.js strutComp fields + main.js call + GLOSSARY

**logger.js** — append-only per Phase 2 D-13 constraint:
- FIELDS array: `'fl_sc', 'fr_sc', 'rl_sc', 'rr_sc'` at end (positions 42–45)
- captureFrame row: `fl.strutComp ?? 0` through `rr.strutComp ?? 0` at end of push
- Doc comment updated: 41 → 45 fields

**main.js** — `updateTravelBars` imported from `./debug.js` and called after `updatePacejkaCurve`, before `renderer.render`.

**docs/GLOSSARY.md** — new "Phase 4.1 Suspension Terms" section with 7 term entries:
1. strut axis — body-local up/down translation axis; rotates with body quaternion
2. strutComp — strut compression from rest length [m], positive = compressed
3. strutCompVel — rate of strut compression [m/s]
4. bump stop — penalty spring at strutComp >= travel; exposed as slider
5. droop stop — penalty spring at strutComp <= 0; fixed constant, no slider
6. body offset — Y shift of mount point in body space for ride-height control
7. hub slider — conceptual 1D slider model; lateral forces transmit instantly via rail constraint
