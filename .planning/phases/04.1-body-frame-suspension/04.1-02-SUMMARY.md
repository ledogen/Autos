# Phase 4.1 Plan 02: Body-Frame Strut Compression — Physics Implementation Summary

**Strut-axis hub ODE + contact-normal split + bump/droop stops: six files modified to replace world-Y hub state with body-frame strutComp/strutCompVel, route X/Z contact normal residual to body, and add travel limit stops**

## What Was Built

### Task 1: Params + state rename (data/ranger.js, src/vehicle.js, src/main.js)

**data/ranger.js** — six new params added after arbStiffnessRear:
- `suspensionTravelFront: 0.12` m, `suspensionTravelRear: 0.14` m — travel before bump stop
- `suspensionBodyOffsetFront: 0.0` m, `suspensionBodyOffsetRear: 0.0` m — mount Y shift (ride-height control, default 0 = no change)
- `bumpStopStiffness: 330000` N/m — ~10x front spring; slider-exposed
- `DROOP_STOP_STIFFNESS: 20000` N/m — fixed constant; ~8.8 mm deflection at hub weight

**src/vehicle.js** — `SPAWN_STATE.hubY`/`hubVy` renamed to `strutComp`/`strutCompVel` with D-01 comments. 0-placeholders overwritten by `computeStaticEquilibrium` at init/reset.

**src/main.js** — four sub-changes:
1. `RANGER_PARAMS._hubNormalXZ` scratch initialized as 4-element array of plain `{x,y,z}` objects
2. `computeStaticEquilibrium` rewritten to return `{bodyY, strutComp}` per D-11: `strutComp[i] = sprung * g / k_S` (verified: ~111 mm at current params)
3. `vehicleState` literal uses `strutComp`/`strutCompVel` instead of `hubY`/`hubVy`
4. R-reset block updated to use `strutComp`/`strutCompVel`; wheel mesh visual (D-07) replaced with full world-space hub position inverse-transformed to body-local via `up.clone().invert()` quaternion

### Task 2: Rewrite src/suspension.js

**`getWheelPosition`** rewritten: derives hub position from `strutComp` along body_down axis:
```
strutLen = L_S - strutComp[corner]
hub = mountWorld + strutLen * body_down
```
Falls back to body_down = (0,-1,0) if `_rotateVector` absent.

**`stepSuspensionSubsteps`** fully rewritten with six sub-changes:
- Guard: checks `strutComp`/`strutCompVel` instead of `hubY`/`hubVy`
- `_hubNormalXZ` zeroed each outer step alongside `_tireFz`/`_suspForceAccum`
- `body_down`/`body_up` derived once per outer step via `_rotateVector`
- Per-corner: full 3D mount velocity, projected onto `body_down` for `mountVelStrut`; hub world position from `strutComp` along `body_down`
- Contact normal split (D-06): `bodyUpDot = dot(c.normal, body_up)`; `Fn_strut = Fn * bodyUpDot` → `tireFz`; residual → `_hubNormalXZ[i]` averaged via `/N`
- Strut-axis hub ODE (D-04/D-05): `F_total = tireFz - k*strutComp - c*strutCompVel + arbF - m_u*g*gravDot + bumpForce + droopForce`
- Body reaction: `suspForce = k*strutComp + c*strutCompVelI` (pre-step velocity for explicit Euler consistency)

**No THREE.js import** — pure-math contract preserved throughout.

### Task 3: src/physics.js + src/logger.js

**src/physics.js** — three changes:
1. `suspBodyForce` now uses `up.clone().multiplyScalar(_suspForceAccum[i])` — strut axis direction (not world-Y)
2. Step 2.6 added: pre-computes XZ force/torque contributions from `_hubNormalXZ`, applies to `totalForce`/`totalTorque` after Step 3 vectors are declared
3. Double-count guard comment updated to document D-06 split architecture

**src/logger.js** — `openInitialCondition` migrated:
- `hubVy = [0,0,0,0]` → `strutCompVel = [0,0,0,0]`
- Hub equilibrium formula replaced with D-11: `strutComp[i] = sprung * g / k_S`

## Deviations from Plan

### Auto-applied architectural choice

**Step 2.6 implementation pattern:** The plan specified inserting Step 2.6 "AFTER substep loop but BEFORE Step 3 loop." However, `totalForce` and `totalTorque` are declared at the start of Step 3. Rather than restructuring Step 3 (declaring totalForce earlier), the implementation pre-computes XZ force/torque contributions into temporary scalar arrays before Step 3, then applies them immediately after `totalForce`/`totalTorque` are declared. This is functionally identical to the plan's intent and avoids restructuring the existing Step 3 pattern.
