<objective>
Make the validated valley-following streaming-anchor trunk the REAL `RoadSystem` routing core, and
retire the failed per-tile west→east A* / hard-grade-block path entirely (the architecture that failed
`.planning/phases/08-road-routing/VERIFICATION.md`).

This plan productionizes the spike-001 prototype's routing engine — `_protoAnchor` (valley-snapped
macro-anchors, D-08), `_protoConnect` (turn-penalty soft-cost A*, D-09), `_protoEdgeCost` (the locked
soft cost model, D-02 REVISED / D-04) — into the canonical network builder, and deletes the old
`_routeTile` / `_seamPoint` / `_deriveEdgeWaypoints` / `_buildTileSpline` / old `_edgeCost` machinery
that forced every 64 m tile to route edge-to-edge with a hard `grade > maxRoadGrade → Infinity` block.

Cost model is LOCKED (D-09):
`edgeCost = wDist·horiz + wAlt·h + wGrade·grade² + wOver·max(0, grade − maxGrade) + wTurn·(Δheading/45°)`.
Defaults: wAlt 0.85, wGrade 400, wOver 8000, maxGrade 0.15, wTurn 120. The over-cap penalty is
**finite/soft** — there is NEVER an `Infinity` edge.

Purpose: a deterministic, infinite, valley-wrapping trunk is the single source of truth the slicing
(08-02), spurs/wiring (08-03), and exit-gate tests (08-04) all build on.
Output: `src/road.js` with a `_streamNetwork(center)` that produces canonical continuous trunk
polylines; `data/ranger.js` carrying the locked D-09 cost defaults; the old per-tile router removed.
</objective>
