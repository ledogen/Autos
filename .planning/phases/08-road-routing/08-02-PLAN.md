    Add public methods to RoadSystem (full JSDoc per PATTERNS.md):

    `init(scene)` — store `this._scene = scene` so debug lines can be added.

    `ensureTile(tileX, tileZ)` — public wrapper over `_getTile`; returns the cached tile object. Used by resolveSpawn (plan 03) and viz to force generation.

    `queryNearest(wx, wz, radiusM = 200)` — iterate currently-cached tile splines (RESEARCH Open Q4: caller pre-generates the region via ensureTile); for each spline sample N points (e.g. spline.getPoints(32)) and track the closest to (wx,wz) in XZ; if the closest is within radiusM, compute the spline parameter and return `{ point, tangent }` using getPointAt/getTangentAt at the nearest arc-length u. Return null if nothing within radius. JSDoc: this is a query helper for resolveSpawn (D-07) and Phase 9 consumption; document the clean stable contract.

    `buildDebugLines()` — for each currently-cached tile spline, build a THREE.Line via the PATTERNS.md `_buildDebugLine` (spline.getPoints(64) → BufferGeometry.setFromPoints → LineBasicMaterial color 0xffaa00), set `line.visible = this._debugVisible`, add to `this._scene`, push to `this._debugLines`. Dispose+clear any existing `_debugLines` first (this is called on rebuild, not per-frame). D-05: centerline lines only — no waypoint/grid geometry.

    `setDebugVisible(enabled)` — store `this._debugVisible = enabled`; if no lines exist yet and enabled, call `buildDebugLines()`; otherwise set `line.visible = enabled` for every line in `_debugLines` (toggle, never dispose). Initialize `this._debugVisible = false` in constructor (clean by default — D-05).

    `invalidateCache()` — per PATTERNS.md: remove each `_debugLines` line from scene, `geometry.dispose()`, reset `_debugLines = []`, `_tileCache.clear()`. JSDoc: called on max-grade/param change; roads are pure fns so next generation recomputes identically (D-03).
