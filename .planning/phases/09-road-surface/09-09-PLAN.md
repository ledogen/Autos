<objective>
Add the REAL exit-gate test for Phase 9: an INTEGRATION test that asserts
`analyticHeight(onRoadXZ) == ribbonVertexY` at real on-road world positions by exercising the
DIVERGENT CALL SITES, not the pure helpers in isolation.

The verifier's meta-claim (09-VERIFICATION.md): the existing harness (test-road-carve.html,
test-road-mesh.html) only asserts the pure functions (carveBlend, crownProfile, sampleCarve,
potholeNoise) in isolation, and the prior height-agreement check only compared sampleHeight vs
analyticHeight (both raw-based, so they trivially agreed). NO test asserted that the visible
ribbon mesh Y equals the physics carve surface Y at the same on-road position. The gate "passed
on paper" while the integrated surfaces disagreed.

This plan adds that test. It builds a real `RoadSystem` + `RoadMeshSystem`, sweeps a real ribbon,
reads the world-space Y of on-road centerline vertices straight out of the BufferGeometry
position attribute, then samples the real physics carve surface at those exact world XZ positions
via `RoadSystem._sampleCarveWorld` (or terrain analyticHeight blend) and asserts the two agree
within tolerance. This is the phase's true exit gate.

Because the surfaces only AGREE after 09-07 (carve-free smoothing + cache) and 09-08 (unified
gradeY source + curvature + pothole arcS) land, this plan depends on both.

Purpose: A failing-then-passing gate that proves the height-agreement violation is closed and
guards against regression.
Output: test/test-road-height-agreement.html (+ harness helper if needed); human-run PASS check.
</objective>
