<objective>
Re-architect the road as a DECAL RIBBON authoritative on top, not a surface that meets the terrain
(supersedes the carve-and-meet approach). This plan delivers the two render-side pillars of the
locked decision:

1. **Ribbon wins depth** — give the shared ribbon material `polygonOffset` (negative factor/units)
   so the opaque ribbon reliably renders over the terrain instead of z-fighting it.
2. **Ribbon edge skirts** — extend each ribbon edge straight down by a fixed `roadSkirtDepth` apron
   (road-dark color) so when terrain falls away below the ribbon edge there is no see-through gap.

The ribbon's TOP surface (driving surface) Y is left exactly as it is — crown + camber + pothole
fold-in from 09-03/08 is untouched. The skirt hangs DOWN from the existing edge vertices; the carve
(Plan 11) guarantees terrain stays below it.

Purpose: Eliminate the shredded / z-fighting / see-through road the user observed in-sim. One
authoritative surface (the ribbon) — exactly how driving games seat roads.
Output: A ribbon that renders cleanly over terrain with a dark apron at each edge, plus three new
tunable params + sliders.
</objective>

  - uLat = (j / CROSS_SEGS - 0.5) * roadWidth  (so j=0 is left edge uLat=-halfWidth, j=CROSS_SEGS is right edge uLat=+halfWidth).
  - vx = posX + rightX * uLat ; vz = posZ + rightZ * uLat ; vy = gradeY + crownY + tiltY + pY.
  - vertex index for (i,j): idx = (i * (CROSS_SEGS + 1) + j) * 3.
  - Asphalt base color RC,GC,BC = 0.15, 0.15, 0.17 (road-mesh.js:177).
