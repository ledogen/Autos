<objective>
P0 — the foundation. Extend the existing per-run camber cache into a unified
`RoadRunProfile` per `runKey` that holds parallel arc-indexed arrays
(`gradeY[]`, `camberRad[]`, `tx[]`, `tz[]`) and exposes a single O(log N)
binary-search sampler `runProfile(arcS, runKey) → { gradeY, camberRad, tx, tz }`.

This is the cure proven by the shipped camber-arc work (`3df47cd`): stop reading
per-fragment values; read everything geometric from ONE continuous per-run
arc-indexed profile by the run-global `arcS`. At any seam both fragments resolve
to the same `arcS`, so anything read by `arcS` is C0 by construction. Every
downstream fix (P2 grade-Y, P3 ribbon frame, P4 camber stitch) hangs off this.

Purpose: One source, one arc domain — height-agreement and seam-continuity by
construction instead of per-consumer reconstruction.
Output: `_buildRunProfile(runKey)` + `runProfile(arcS, runKey)` in src/road.js.
</objective>
