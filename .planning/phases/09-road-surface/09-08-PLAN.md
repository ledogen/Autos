sampler (`sampleDesignGradeAt`), this plan switches the divergent call sites onto it.

CR-01 (BLOCKER): The visible ribbon (road-mesh.js sweepRibbon) reads its base elevation from
the SMOOTHED design grade (designGradeY[i]). Both physics carve sites read base elevation from
the RAW routing spline Y (road.js:1346 `designY = nr.point.y`, terrain.js:857 same). The truck
drives a different surface than the asphalt it visibly sits on. Fix: both carve sites derive the
base gradeY from `sampleDesignGradeAt(spline, nr.arcS, rawHeightWorld, params)` — the SAME
smoothed source the ribbon uses — instead of `nr.point.y`. This is the locked
"ribbon-as-skin over one shared smoothed height field" decision.

CR-02 (PARTIAL): Camber magnitude uses two estimators — mesh kappa = dtLen/(du·arcLen) with
du≈0.02 normalized-u (road-mesh.js:139), carve sites kappa = dtLen/eps with eps=2.0 m world-space
(road.js:1371, terrain.js:881). Fix: extract ONE shared `signedCurvature` function and call it
with identical arguments at all three sites so banking matches at the lateral extremes.

`arcSOffset + u·arcLen` (road-mesh.js:219-220), carve sites use laterally-offset `nr.arcS`
(road.js:1388-1389, terrain.js:895-896). Fix: snap all sites to a single centerline arcS before
calling roadQuality so severity (hence pothole Y) matches at stretch/tier boundaries.

Purpose: Make the physics surface and the visible ribbon agree by construction — the phase exit
gate (mesh-Y == physics-Y at on-road positions).
