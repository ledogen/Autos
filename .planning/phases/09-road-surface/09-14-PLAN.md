<objective>
Rate-limit road camber along the arc so banking eases in and out with a bounded d(camber)/ds. Eliminate the bump-outside/dip-inside on tightening turns AND the clamp-flip spike at curvature zero-crossings. Apply the SAME rate-limit consistently in the visual ribbon (road-mesh.js sweepRibbon) and the physics camber (road.js _sampleCarveWorld) so visual == physics. Add a tunable rate param + slider; keep the +/-6 degree clamp.

Purpose: camber is currently `clamp(camberStrength * signedKappa, +/-6 degrees)` evaluated instantaneously per section — it spikes on tight turns (harness caught 12.76 deg/m) and flips sign abruptly at curvature zero-crossings. The fix is a low-pass over the camber angle along arc length.

Output: a smooth, bounded-rate camber profile shared by mesh + physics, with the harness extended to numerically prove d(camber)/ds stays under threshold.
</objective>

    - Input: `rawAnglesRad` = Float32Array|number[] of per-sample raw (already clamped) camber angles in radians, in arc order; `arcStepM` = arc-length spacing between samples (metres); `maxRateRadPerM` = maximum allowed |d(angle)/ds| (radians per metre).
    - Algorithm: a symmetric forward+backward slew-rate limiter. Forward pass from i=1..N-1: `out[i] = clamp(raw[i], out[i-1] - maxRateRadPerM*arcStepM, out[i-1] + maxRateRadPerM*arcStepM)`, seeded `out[0] = raw[0]`. Backward pass from i=N-2..0: `out[i] = clamp(out[i], out[i+1] - maxRateRadPerM*arcStepM, out[i+1] + maxRateRadPerM*arcStepM)`. The two-pass approach gives a symmetric ease-in/ease-out and prevents the clamp-flip spike at zero-crossings (the slew bound forces the sign change to ramp through zero over multiple samples).
    - Degenerate guards: if N==0 return empty same-type array; if `arcStepM <= 0` or `maxRateRadPerM <= 0` return a copy of the input unchanged; never produce NaN.
    - Return a Float32Array (or new array) of the same length — do NOT mutate the input.
    - Pure function, deterministic (D-16). Do NOT add THREE imports.
  </action>
