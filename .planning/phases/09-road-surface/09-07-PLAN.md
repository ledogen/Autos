<objective>
Fix CR-04 (the structural double-count + stale cache) that underlies the height-agreement
gate failure. Two coupled defects:

1. `_smoothDesignGrade` is fed `_terrainRef = (x,z) => terrainSystem.analyticHeight(x,z)`
   (main.js:840), which is CARVE-INCLUSIVE. On-ribbon, analyticHeight returns
   `gradeY = rawSpline + crown + camber + pothole` (blendW=1), so the 50 m smoothing window
   averages a surface that ALREADY contains crown/camber/pothole. sweepRibbon then ADDS
   crown/camber/pothole AGAIN (road-mesh.js:243-260) → a structural double-count of the
   0.05 m crown plus the camber/pothole terms in the visible ribbon Y.

2. `_designGradeCache` (road.js:1442, a WeakMap keyed by spline object) is only invalidated
   when the `window` value changes. `debouncedRoadSurfaceRebuild` (main.js:261-275) clears
   ribbon tiles and re-bakes carve tables on crownHeight / terrainAmplitude / camberStrength
   slider changes, but the spline objects persist, so the WeakMap returns STALE smoothed grade.

This plan makes `_smoothDesignGrade` consume a carve-free raw-terrain sampler and adds an
explicit cache-invalidation hook wired into the surface-param rebuild path. It also exposes
a single arc-keyed smoothed-grade lookup helper on RoadSystem — the shared elevation source
that plan 09-08 will call from both carve sites.

This plan does NOT yet change the carve sites (road.js _sampleCarveWorld / terrain.js
_buildCarveTable still read raw nr.point.y) — that is 09-08, which depends on this plan.

Purpose: Remove the double-count and stale-cache defects so the unified gradeY source built
in 09-08 reads a clean, correct, cache-coherent smoothed profile.
Output: Carve-free sampler, cache invalidation, shared arc-keyed grade lookup.
</objective>
