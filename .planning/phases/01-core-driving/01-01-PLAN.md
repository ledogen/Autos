    Sections required by D-02 — each section must be populated, no placeholders:

    1. `## Coordinate System` — Three.js Y-up right-handed world space. State explicitly: +X = right, +Y = up, -Z = car forward at heading 0 (heading 0 means the car points down the -Z axis). State that no SAE / ISO automotive conversion layer exists anywhere in the codebase; physics math uses Three.js axes directly.

    2. `## Named Vectors (Body Space)` — Three named unit vectors used everywhere physics is written. In body-space:
       - `forward` = `(0, 0, -1)` rotated by `vehicleState.quaternion` via `THREE.Vector3.applyQuaternion(quaternion)`
       - `right`   = `(1, 0, 0)` rotated similarly
       - `up`      = `(0, 1, 0)` rotated similarly
       Document that any time a module computes "the direction the car is pointing" it MUST construct that vector from this rotation, NEVER from `Math.atan2(velocity.x, velocity.z)` or from a stored Euler `heading` scalar.

    3. `## Sign Conventions`
       - `### Slip Angle` — Define as the angle between the wheel's heading direction (its `forward` vector after steer rotation) and the velocity vector at the wheel contact patch. Positive sign convention: counter-clockwise when viewed from above in the Y-up right-hand system (i.e., positive slip angle corresponds to the contact patch velocity pointing to the wheel's left). State that Phase 1 does not use slip angle math (D-08 — velocity damping only); the convention is documented now for Phase 3 Pacejka.
       - `### Torque` — Positive torque about an axis follows the right-hand rule with the axis as the thumb. World-frame torques applied in `physics.js` accumulate into `angularVelocity` (also world-frame). Yaw torque about +Y produces counter-clockwise yaw viewed from above.
       - `### Steering Angle (Scalar, NOT Euler)` — `steerAngle` is a scalar in radians stored on `vehicleState`; positive = steer left (counter-clockwise viewed from above, consistent with positive yaw about +Y). It is NOT an Euler angle for body rotation — it is a separate per-frame input scalar consumed by Ackermann geometry. Body rotation remains a quaternion at all times.

    4. `## Quaternion Integration Convention` — State exactly how `vehicleState.quaternion` is integrated each physics step:
       - `angularVelocity` is stored in WORLD frame (rad/s).
       - Each step: build `dq = THREE.Quaternion.setFromAxisAngle(omega.clone().normalize(), |omega| * dt)`.
       - Update by left-multiply (world-frame convention): `q.premultiply(dq)`.
       - Always call `q.normalize()` immediately after, every step, to prevent drift.
       - If `|omega| < 1e-10`, skip the update (no rotation this frame).
       Document the rationale: a body-frame convention would use `q.multiply(dq)` instead; this project uses world-frame `premultiply`.

    5. `## Term Definitions`
       - `### Slip Angle` — Angle between wheel heading and contact patch velocity. Unit: radians. Phase 3 will use this in Pacejka.
       - `### Contact Patch Velocity` — Velocity vector at the location where the tire meets the ground, computed as `vehicleState.velocity + vehicleState.angularVelocity × (wheelContactPoint - vehicleState.position)`. Decomposed into longitudinal (along wheel's `forward`) and lateral (along wheel's `right`) components for tire force calculation. Unit: m/s.
       - `### Ackermann Geometry` — Steering geometry where the inner wheel rotates by a sharper angle than the outer wheel to minimize tire scrub during a low-speed turn. The two wheels track concentric arcs whose center lies on the extended rear-axle line. Phase 1 uses the exact sin/cos form documented in RESEARCH.md §Pattern 5 with `L = 2.85 m` and `T = 1.46 m`. Unit: radians (input and output).

    6. `## Wheel Index Convention` — Repeat the mapping that every wheel-indexing module will use:
       - `0 = FL (Front Left)`
       - `1 = FR (Front Right)`
       - `2 = RL (Rear Left)`
       - `3 = RR (Rear Right)`

    7. `## Deferred to Phase 3 / Phase 4` — Explicitly list terms NOT defined yet: Pacejka B/C/D/E coefficients, friction circle, longitudinal slip ratio, omega_wheel, spring stiffness, damping coefficient, normal force load transfer, ride height. Each is a one-liner referencing the future phase that will own its definition.

    Do NOT reference `references/backup11.html`, `references/backup12.html`, or `references/backup12alt.html` anywhere in the document (D-03). The conventions above are derived from first principles + RESEARCH.md + PROJECT.md only (D-04).
